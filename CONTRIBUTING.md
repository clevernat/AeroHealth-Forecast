# Contributing to AeroHealth Forecast

Thank you for your interest in contributing to AeroHealth Forecast! This project serves an important public health mission, and your contributions can help improve respiratory health outcomes for people everywhere.

## üåü How You Can Contribute

### Reporting Bugs
- Check if the bug has already been reported in [Issues](https://github.com/yourusername/aerohealth-forecast/issues)
- If not, create a new issue with:
  - Clear title and description
  - Steps to reproduce
  - Expected vs actual behavior
  - Screenshots if applicable
  - Browser/device information

### Suggesting Enhancements
- Open an issue with the "enhancement" label
- Describe the feature and its benefits
- Explain how it aligns with the public health mission
- Provide examples or mockups if possible

### Code Contributions

#### Getting Started
1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/aerohealth-forecast.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development server: `npm run dev`

#### Development Guidelines
- **Code Style**: Follow the existing code style (enforced by ESLint)
- **TypeScript**: Use proper typing; avoid `any` types
- **Components**: Keep components focused and reusable
- **Accessibility**: Ensure all features are accessible (WCAG 2.1 AA)
- **Performance**: Optimize for mobile and slow connections
- **Testing**: Test on multiple browsers and devices

#### Commit Messages
Use clear, descriptive commit messages:
- `feat: Add location search functionality`
- `fix: Correct AQI calculation for PM2.5`
- `docs: Update README with deployment instructions`
- `style: Format code with Prettier`
- `refactor: Simplify pollen data processing`

#### Pull Request Process
1. Update documentation if needed
2. Ensure the build passes: `npm run build`
3. Test thoroughly in development: `npm run dev`
4. Create a pull request with:
   - Clear description of changes
   - Reference to related issues
   - Screenshots for UI changes
   - Testing notes

### Documentation
- Improve README.md
- Add code comments for complex logic
- Update educational content about pollutants/allergens
- Create tutorials or guides

### Design
- Improve UI/UX
- Create mockups for new features
- Enhance accessibility
- Optimize for mobile devices
